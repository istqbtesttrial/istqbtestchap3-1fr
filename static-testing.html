Qu'est-ce que le Test Statique ?

Le test statique est une technique permettant de détecter les défauts dans les produits de travail logiciels sans exécuter le code. Il aide à identifier les défauts tôt, souvent à un coût moindre, en examinant les exigences, les conceptions et le code.

Exemple Interactif : Trouvez les Défauts

Voici un exemple de spécification des exigences. Cliquez sur les zones où vous pensez qu'il y a des défauts :

  • 1. Le système doit permettre aux utilisateurs de se connecter avec leur nom d'utilisateur et leur mot de passe.
  • 2. Le système doit permettre aux utilisateurs de réinitialiser leur mot de passe par email ou SMS.
  • 3. Le système doit fournir des messages d'erreur détaillés qui révèlent des informations internes sur le système.
  • 4. Le système doit stocker les mots de passe en texte clair dans la base de données.
